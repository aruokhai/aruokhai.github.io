Merkle Index

Patients' data are usually scattered in different health care providers' record systems, with no effective way to identify patients within healthcare providers or across healthcare providers.These isolated silos of patient data have huge potential to improve the health of patients. The potential can be realised only if there is a way to link records in disparate health care provider silos with a global identifier. Such an identifier solution would provide two use cases: the first is to link disparate healthcare data about a patient under an identity, and the second is to grant access to respective healthcare providers. A solution like this would further the goal of a personal health record system..

Solution

MerkleIndex, serving as a comprehensive Global Index Solution, leverages both OAUTH 2 and SUBSTRATE to create an authorization system with a dual natureâ€”global in scope yet locally processed. This unique approach combines OAUTH2 for patient recognition across institutions and token generation for resource access. SUBSTRATE blockchain technology is employed to facilitate secure sharing of these tokens among interested parties, particularly healthcare providers. In this decentralised network, each healthcare provider operates as a substrate node host and employs a token system for node coordination. To maintain network integrity and minimise spamming, tokens are consumed during each transaction broadcast but are regularly replenished.

Design Choice

  • Asynchronous Operation: The system operates in an asynchronous mode, enhancing efficiency and responsiveness.
  • Patient Trustworthiness: Patients are not required to place trust in third-party services, ensuring data security and patient autonomy.
  • Global Patient Identification: Each patient is associated with a global blockchain address (Pubkey), enabling seamless identification and access across the network.
  • Healthcare Provider Node Operators: All healthcare providers play the role of blockchain node operators, ensuring the distributed and secure nature of the network.
